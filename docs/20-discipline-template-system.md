# 20. Система шаблонов дисциплин

## Концепция

Трёхуровневая модель позволяет SportOS поддерживать любой вид спорта без хардкода дистанций:

    Вид спорта (Sport)              ← Каталог, фиксированный
        └─ Шаблон дисциплины (Template) ← Правила спорта, фиксированный
             └─ Дисциплина мероприятия  ← Настраивается организатором

### Что фиксировано (Шаблон)
- Название типа (Скиджоринг, Нарты, Классика, МТБ XC...)
- Оборудование и ограничения (кол-во собак, тип лыж, тип велосипеда)
- Специфические правила (штрафы, обязательное снаряжение)

### Что настраивает организатор
- **Длина круга** (м) — точная длина трассы (напр. 3750 м)
- **Кругов** — количество кругов
- **Общая дистанция** = длина круга × кругов (авто, напр. 3750 × 8 = 30 000 м = 30 км)
- **Категории** — чекбоксы из доступных для шаблона + возрастные группы
- **Тип старта** — раздельный / масс-старт / волна / гандикап
- **Интервал старта** (для раздельного)
- **Cutoff** — максимальное время
- **Стоимость** — цена регистрации

> **Точная дистанция в протоколе.** В итоговом протоколе всегда фиксируется точная дистанция (напр. «30.000 км»), а не «30 км». Это необходимо для подсчёта средней скорости, темпа, и сравнения результатов между мероприятиями.

## Система категорий

Организатор может включать любые категории из доступных в шаблоне + настраивать возрастные группы:

### Гендерные категории
| ID | Название | Описание |
|----|---------|----------|
| `M` | Мужчины | 18+ |
| `F` | Женщины | 18+ |
| `JM` | Юниоры | 14-17 лет |
| `JF` | Юниорки | 14-17 лет |
| `KIDS` | Дети | до 14 лет |

### Возрастные группы (настраиваемые)
Организатор добавляет возрастные группы из типовых или создаёт свои:

| ID | Название | Возраст |
|----|---------|--------|
| `M35` | Мужчины 35 | 35-39 |
| `M40` | Мужчины 40 | 40-44 |
| `M45` | Мужчины 45 | 45-49 |
| `M50` | Мужчины 50 | 50-54 |
| `M55` | Мужчины 55 | 55-59 |
| `M60` | Мужчины 60+ | 60+ |
| `F35` | Женщины 35 | 35-39 |
| `F40` | Женщины 40 | 40-44 |
| `F45` | Женщины 45 | 45-49 |
| `F50` | Женщины 50+ | 50+ |
| `VET` | Ветераны | определяется правилами вида спорта |

> **Настройка возрастных групп.** Организатор может: включить/выключить стандартные группы, изменить диапазоны возрастов, создать кастомную группу (напр. «Мальчики 8-10»).

> **Абсолютный зачёт vs. Групповой.** Протокол может формироваться в абсолютном зачёте (все вместе) И в групповом (по категории). Одновременно.

## Мультиэтапные дисциплины (Multi-Stage)

Для мероприятий где один участник проходит несколько этапов разных видов спорта:

```
Мультиэтапная дисциплина
  ├─ Этап 1: swim.openwater  → 1.5 км
  ├─ Транзитная зона T1
  ├─ Этап 2: cycle.road      → 40 км
  ├─ Транзитная зона T2
  └─ Этап 3: run.long         → 10 км
```

### Параметры мультиэтапной дисциплины:
- **Этапы** — список шаблонов с дистанциями (порядок важен)
- **Транзитные зоны** — количество и время лимита
- **Общий cutoff** — максимальное суммарное время
- **Cutoff на этап** — ограничение по каждому этапу
- **Общий зачёт** — сумма времён всех этапов + транзиции

### Правила хронометража транзиций (World Triathlon / ITU)

По правилам World Triathlon время транзиции **входит в общее время** гонки — часы никогда не останавливаются:

```
Общее время = Swim + T1 + Bike + T2 + Run
```

В протоколе фиксируются **5 сегментов** раздельно:

| Этап | Начало | Конец |
|------|--------|-------|
| Swim | Старт | Выход из воды |
| T1 | Выход из воды | Mount Line (сел на вел) |
| Bike | Mount Line | Dismount Line (слез с вела) |
| T2 | Dismount Line | Выход из транзитной зоны |
| Run | Выход из T2 | Финиш |

> **В SportOS:** для мультиэтапных дисциплин — разметка транзитных зон с матами/NFC тегами, автоматический split на этапы, и запись каждого сегмента в протокол.

## Точная дистанция и расчёты в протоколе

В итоговом протоколе фиксируется **точная дистанция** в метрах (не округлённая). Это необходимо для:

| Показатель | Формула | Пример (30.000 км, 1:45:30) |
|---|---|---|
| Средняя скорость (км/ч) | дистанция / время | 17.06 км/ч |
| Темп (мин/км) | время / дистанция | 3:31 мин/км |
| Сплиты по кругам | время круга N | Круг 1: 12:45, Круг 2: 13:10... |
| Отставание | время − время лидера | +2:15 |

### Пример строки протокола

    Дисциплина: Скиджоринг (1 собака)
    Дистанция: 30.000 км (круг 3750м × 8 кругов)
    Категория: Мужчины 35-39

    №  | BIB | ФИО          | Кр.1   | Кр.2   | ... | Кр.8   | Итог    | km/h  | Отстав.
    1  | 07  | Петров А.А. | 12:45  | 13:10  | ... | 14:02  | 1:45:30 | 17.06 | —
    2  | 12  | Сидоров Б.  | 12:50  | 13:25  | ... | 14:30  | 1:47:45 | 16.69 | +2:15

## Модель данных (концепт)

```dart
class SportType {
  String id;           // 'sled', 'skiing', 'trail'...
  String name;         // 'Ездовой спорт'
  String icon;         // '🐕'
  int color;           // 0xFF1565C0
  List<DisciplineTemplate> templates;
}

class DisciplineTemplate {
  String id;           // 'sled.skijor-1'
  String name;         // 'Скиджоринг (1 собака)'
  String sportId;      // 'sled'
  int? dogCount;       // null для не-собачьих
  String? equipment;   // 'Лыжи + потяг'
  List<String> availableCategories;  // ['М', 'Ж', 'Юн', 'Юнк', 'Вет']
  List<String> availableAgeGroups;   // ['M35', 'M40', 'M45', 'M50', 'F35', 'F40'...]
  Map<String, dynamic>? extraFields; // D+ для трейла, стиль для лыж...
  bool isMultiStage;   // true для триатлона
  List<StageTemplate>? stages;       // этапы мультиспортивной
}

class EventDiscipline {
  String id;           // UUID
  String templateId;   // 'sled.skijor-1'
  int lapLengthMeters; // 3750  ← длина круга
  int laps;            // 8     ← кол-во кругов
  double get totalDistanceKm => lapLengthMeters * laps / 1000.0; // 30.000 км (авто)
  List<String> activeCategories; // ['М', 'Ж', 'M35', 'M40', 'Юн']
  List<AgeGroup> ageGroups;      // кастомные возрастные группы
  String startType;    // 'individual' | 'mass' | 'wave' | 'handicap'
  int? intervalSec;    // 30 (null для масс-старт)
  Duration cutoff;     // 2:00:00
  int priceRub;        // 2500
  String get displayName => '${template.name} ${totalDistanceKm}км';
  // → "Скиджоринг-1 30.000км"
}

class AgeGroup {
  String id;       // 'M35'
  String name;     // 'Мужчины 35'
  String gender;   // 'M' | 'F'
  int ageFrom;     // 35
  int ageTo;       // 39  (null = без верхней границы)
}
```
