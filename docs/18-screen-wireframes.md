## 4. Регистрация на мероприятие

### Дизайн-решения

| Решение | Описание |
|---|---|
| **Минимум трения** | Регистрация = 2 шага за 15 секунд |
| **Собака не блокирует** | Можно выбрать позже, хоть за час до старта |
| **Оплата не блокирует** | Место бронируется, таймер на оплату |
| **Таймаут брони** | Настраивается организатором, дефолт 48ч |
| **Промокод** | На шаге 1, мгновенный пересчёт суммы |

### Подтверждение оплаты (атлет → организатор)

```
Атлет: «Я оплатил» → опционально фото чека
  → push организатору: «Петров оплатил, проверьте»
  → организатор: [✅ Подтвердить] [❌ Отклонить]

Фото чека: хранится в Supabase Storage
  - Сжимается при загрузке до 300 KB
  - Удаляется через 90 дней
  - Факт оплаты (Payment record) хранится бессрочно
```

## 7. Управление заявками

### Дизайн-решения

| Решение | Описание |
|---|---|
| **Массовые действия** | Выбрать несколько → подтвердить оплату / снять бронь разом |
| **Ручное добавление** | Организатор может добавить атлета вручную |
| **GhostProfile** | Атлет без аккаунта → ghost-профиль (имя + телефон) |

| **Auto-expire** | Просроченные брони снимаются автоматически |

## 8. Мандатная комиссия

### Дизайн-решения

| Решение | Описание |
|---|---|
| **Конфигурируемый чек-лист** | Организатор настраивает пункты. Можно отключить всё |
| **Подсказки из профиля** | Сроки вакцинаций, страховки подтягиваются автоматически |
| **⚠️ не блокирует** | Просроченный документ = предупреждение, решает судья |
| **Допустить без всех ✅** | Кнопка «Допустить» активна всегда — решение за судьёй |
| **Командная проверка** | Отдельный режим для представителя команды |
| **Порядок** | Мандатная + ветконтроль ДО жеребьёвки |

### Настройка организатором

## 11. Финиш (3 режима)

### Дизайн-решения

| Решение | Описание |
|---|---|
| **Дисциплино-агностичный** | Финиш ловит всех, цвет дисциплины на плитке |
| **3 режима** | Полный, Только отсечки, Назначение |
| **Ошибочная отсечка** | Свайп влево → удалить пустую. С BIB → long press + подтверждение |
| **Ручная коррекция** | Long press → изменить время + причина → audit log |
| **Одновременный финиш** | Судья тапает быстро; можно вручную задать одинаковое время |
| **Мастер + контрольный** | Все устройства ловят время параллельно; мастер — основной |
| **Объединение отсечек** | Система сравнивает линии, предлагает дополнить пропущенные |

### Режим «Полный» (1 человек — всё сам)

Для маленьких гонок (5-15 участников).

### Дизайн-решения

| Решение | Описание |
|---|---|
| **Видит всё** | Старты, трасса, финиши — всё в одном экране |

## 19. Ветконтроль

### Дизайн-решения

| Решение | Описание |
|---|---|
| **Два режима** | До старта (осмотр) и после финиша (чип) |
| **Настраивается** | Организатор включает/выключает каждый режим |
| **Сканер чипа** | Bluetooth-сканер или ручной ввод номера |
| **Подмена** | Если чип не совпал → нарушение → DSQ → главному судье |
| **Проверяются все** | После финиша — все финишировавшие |

## 22. Команда и роли

### Дизайн-решения

| Решение | Описание |
|---|---|
| **Роли = наборы прав** | Встроенные пресеты + кастомные роли |
| **Название свободное** | «Главный судья» = судья с расширенными правами |
| **Несколько ролей** | Один человек → несколько ролей |
| **Приглашение** | Поиск + ссылка + QR-код |
| **PIN-доступ** | Гостевой маршал через браузер без приложения |

### Атомарные права

```
Заявки:
  ☐ Просмотр │ ☐ Подтверждение оплаты │ ☐ Ручное добавление

Мандатная:
  ☐ Проверка документов │ ☐ Финальный допуск

Жеребьёвка:
  ☐ Создание/редактирование стартового списка

Финиш:
  ☐ Отсечки │ ☐ Назначение BIB │ ☐ Коррекция времени

Старт:
  ☐ Старт атлетов │ ☐ DNS

Трасса:
  ☐ Маршал (прохождение, DNF, SOS)

Штрафы:
  ☐ Фиксировать нарушение (→ судье)
  ☐ Назначать штраф
  ☐ Отменять штраф

Протесты:
  ☐ Рассматривать протесты

Ветконтроль:
  ☐ Осмотр собак │ ☐ Чип-проверка

Диктор:
  ☐ Просмотр live-данных

Управление:
  ☐ Редактирование мероприятия │ ☐ Управление командой
```

| Решение | Описание |
|---|---|
| **Всё настраивается** | Организатор определяет правила каждого дня |
| **Порядок дня 2** | Обратный к дню 1 / тот же / новая жеребьёвка |
| **Итог** | Сумма / лучший / среднее — настраивается |
| **DNF в один день** | Вылетает из общего зачёта, но результат за другой день сохраняется |
| **Ветконтроль** | Особенно важен на 2-й день (проверка чипа = защита от подмены) |
