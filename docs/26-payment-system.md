# 26. Платёжная система SportOS

> Архитектура приёма платежей: от ручного перевода до онлайн-эквайринга.

---

## 1. Концепция

Организаторы имеют разный юридический статус. Платёжная система адаптируется:

| Уровень | Статус организатора | Способ приёма | Комиссия |
|---|---|---|---|
| **Tier 0** | Физлицо (без ИП) | Реквизиты + чек | 0% |
| **Tier 1** | ИП / самозанятый | СБП + реквизиты | 0.4–0.7% |
| **Tier 2** | ООО / ИП | Tinkoff Acquiring / YooKassa | 1.5–3.5% |

---

## 2. Tier 0 — Ручная оплата (MVP, текущая)

```
Участник → видит реквизиты (карта/счёт) → переводит в банковском приложении → загружает скриншот чека → организатор подтверждает
```

### Настройки организатора
- Номер карты (маскированный: 4276 **** **** 1234)
- ФИО получателя
- Банк получателя
- Таймаут брони (12/24/48/72 часа)
- Инструкция для участника (текст)

### UX для участника
1. Нажимает «Зарегистрироваться»
2. Видит сумму + реквизиты
3. Кнопка «Скопировать номер карты»
4. Переводит в своём банковском приложении
5. Возвращается → загружает фото/скриншот чека
6. Статус: «⏳ Ожидает подтверждения»
7. Организатор проверяет → «✅ Оплачено» / «❌ Отклонено»

### Когда использовать
- Малые мероприятия (до ~50 участников)
- Организатор — физлицо без ИП
- Нет расходов на эквайринг

---

## 3. Tier 1 — СБП / Платёжные ссылки

```
Участник → нажимает «Оплатить по СБП» → deeplink в банковское приложение → оплата → webhook → статус обновлён
```

### Требования к организатору
- ИП или самозанятый
- Подключённый СБП для приёма платежей (через банк)
- Или: генерация QR-кода СБП вручную

### Интеграция
- **СБП C2B API** (НСПК) — для автоматических платежей
- Или **Тинькофф / Сбер для бизнеса** — генерация платёжных ссылок через API

### Поток
1. Организатор подключает СБП в настройках (ИНН + merchant ID)
2. При регистрации участника система генерирует уникальную ссылку СБП
3. Участник нажимает → открывается банковское приложение
4. После оплаты — webhook → статус автоматически обновляется
5. Организатор видит поступление в реальном времени

### Комиссия
- СБП C2B: **0.4–0.7%**, макс 1500₽ за транзакцию
- Для организатора это ~10–35₽ при стартовом взносе 2500₽

---

## 4. Tier 2 — Онлайн-эквайринг (полноценный)

```
Участник → «Оплатить» → WebView/Safari → форма оплаты → карта/SberPay/TinkoffPay/MirPay → 3D-Secure → webhook → ✅
```

### Поддерживаемые провайдеры

| Провайдер | Комиссия | Методы оплаты | API |
|---|---|---|---|
| **Tinkoff Acquiring** | 1.5–2.9% | Карты, SberPay, TinkoffPay, MirPay, СБП | REST + SDK |
| **YooKassa** | 2.8–3.5% | Карты, ЮMoney, SberPay, QIWI, СБП | REST |
| **CloudPayments** | 2.3–2.7% | Карты, Apple Pay, Google Pay, СБП | REST + SDK |
| **Робокасса** | 2.3–5% | 40+ методов | REST |

### Рекомендация: Tinkoff Acquiring
- Низкая комиссия
- Нативный Flutter SDK
- SberPay + TinkoffPay + MirPay + СБП в одном
- Быстрое подключение для ИП

### Поток
1. Организатор заполняет данные ИП/ООО в настройках
2. Подключается через API провайдера (Tinkoff Merchant ID)
3. При регистрации → генерируется Payment Session
4. Форма оплаты открывается через SafariViewController (iOS) / Custom Tab (Android)
5. После оплаты → webhook → статус обновлён
6. Деньги на расчётный счёт организатора (T+1 или T+3)

## 6. Членские взносы клубов

Та же система применяется для клубных взносов:

| Параметр | Значение |
|---|---|
| Тип платежа | Членский взнос клуба |
| Автоплатёж | Да (через Tier 2, рекуррентные платежи) |
| Период | Месяц / Квартал / Год |
| Скидки | Семейные пакеты, скидка за год |

### Поток
1. Клуб настраивает членский взнос (сумма, период)
2. Участник → «Вступить в клуб» → оплата по выбранному Tier
3. При Tier 2: автоплатёж (рекуррентный) через Tinkoff/YooKassa
4. При Tier 0/1: ручной перевод, напоминания в Telegram-боте

## 8. Безопасность платежей

- PCI DSS: SportOS **НЕ хранит** данные карт — это делает эквайер
- Все платёжные формы — через SafariVC / Custom Tab (sandbox)
- Webhook подписи — HMAC-SHA256 от провайдера
- Чеки по 54-ФЗ — генерирует эквайер (для Tier 2)
- Tier 0: организатор сам отвечает за налоговую отчётность
