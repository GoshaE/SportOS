# 21. Система клубов

## Концепция

Клуб — организационная единица объединяющая спортсменов, тренеров и организаторов.
Клуб может проводить мероприятия от своего имени и управлять членством.

    Пользователь
      ├─ Личный профиль (спортсмен, собаки, результаты)
      ├─ Клуб 1: «Быстрые лапы» (участник)
      ├─ Клуб 2: «Trail Ural» (тренер)
      └─ Свои мероприятия (от личного имени)

> **Финансы в приложении** — только учёт и контроль. Оплата происходит вне приложения (перевод, наличные, сайт клуба). Это сознательное решение, чтобы не платить 30% комиссии App Store / Google Play.

### Публичная страница клуба
Видна всем пользователям:
- Информация о клубе
- Предстоящие мероприятия клуба
- Результаты и достижения
- Кнопка **«Подать заявку»**

## Роли в клубе

### Встроенные роли

| Роль | Управл. клубом | Участники | Мероприятия | Финансы | Тренировки |
|------|:-:|:-:|:-:|:-:|:-:|
| **Владелец** | ✅ | ✅ | ✅ | ✅ | ✅ |
| **Администратор** | ⚙️ | ✅ | ✅ | ✅ | — |
| **Тренер** | — | 👁 | ✅ | — | ✅ |
| **Участник** | — | — | 👁 | — | 👁 |

- **Владелец** — создатель клуба. Может передать владение. Удалить клуб.
- **Администратор** — все права кроме удаления клуба и передачи владения.
- **Тренер** — видит своих спортсменов, может заявлять на мероприятия, составлять планы.
- **Участник** — базовое членство.

### Кастомные роли (v2)

Организатор клуба создаёт роль и назначает набор прав из списка:

| Право | Описание |
|-------|----------|
| `members.view` | Просмотр списка участников |
| `members.manage` | Приглашение / исключение |
| `events.create` | Создание мероприятий от клуба |
| `events.manage` | Управление мероприятиями |
| `finances.view` | Просмотр финансов (взносы, баланс) |
| `finances.manage` | Отметка оплаты, создание взносов |
| `training.view` | Просмотр тренировочных планов |
| `training.manage` | Составление тренировок |
| `club.settings` | Настройки клуба (название, лого) |

> **Простой подход:** В MVP — только 4 встроенные роли. Кастомные роли — когда станет понятно, какие нестандартные комбинации нужны пользователям.

## Членство

### Жизненный цикл

    Подача заявки → Рассмотрение (адм.) → Принят → Активный → (Выход / Исключение)

### Членские взносы

| Параметр | Описание |
|----------|----------|
| Тип | Ежегодный / ежемесячный / разовый / бесплатно |
| Сумма | Разная для категорий (взрослый, ребёнок, семейный) |
| Статус | ✅ Оплачен / ⏳ Ожидает / ❌ Просрочен / 🎁 Льготный |
| Контроль | Администратор отмечает оплату вручную |

> Оплата происходит ВНЕ приложения. В SportOS — только учёт: кто оплатил, кто нет, напоминания.

### Привилегии членства
- Скидка на мероприятия клуба (через промокоды)
- Доступ к клубным мероприятиям (закрытые старты)
- Клубная экипировка (заявки через приложение)

## Мероприятия и клубы

### Два режима создания

| Режим | Описание |
|-------|----------|
| **От имени клуба** | «Быстрые лапы» проводят «Ночная гонка» — лого клуба, бренд |
| **Личное** | Иванов А. проводит «Тренировочный старт» — без привязки к клубу |

### Типы мероприятий клуба

| Тип | Доступ | Пример |
|-----|--------|--------|
| **Открытое** | Все | «Ночная гонка на УТЦ Кавголово» |
| **Клубное** | Только члены клуба | «Контрольный старт» |
| **Межклубное** | Несколько клубов | «Дружеский матч: Быстрые лапы vs Trail 66» |

## Федерации и регионы (заготовка)

Пока не реализуем, но закладываем в архитектуру:

    Федерация (РФ)
      ├─ Округ (УФО, ЦФО, СФО...)
      │    └─ Регион (Свердловская обл.)
      │         └─ Клуб «Быстрые лапы»
      └─ Мероприятие: Чемпионат России (участники от клубов/регионов)

> **В коде:** у клуба есть необязательное поле `regionId` и `federationId`. Пока не используем, но данные готовы.

Если выбран клуб → логотип, название клуба в карточке мероприятия.

## Модель данных (концепт)

```dart
class Club {
  String id;
  String name;
  String? logoUrl;
  String city;
  String? description;
  List<String> sportIds;     // ['sled', 'trail']
  String ownerId;            // userId создателя
  String? regionId;          // заготовка для федераций
  String? federationId;      // заготовка
  MembershipConfig membership;
}

class MembershipConfig {
  String type;        // 'annual' | 'monthly' | 'one_time' | 'free'
  Map<String, int> fees; // {'adult': 3000, 'child': 1500, 'family': 5000}
}

class ClubMember {
  String id;
  String clubId;
  String userId;
  String role;        // 'owner' | 'admin' | 'coach' | 'member' | custom
  String status;      // 'active' | 'pending' | 'suspended'
  String feeStatus;   // 'paid' | 'pending' | 'overdue' | 'exempt'
  DateTime joinedAt;
  DateTime? feePaidUntil;
}
```
