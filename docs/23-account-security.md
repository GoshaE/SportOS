## 1. Авторизация

### Способы входа

| Способ | MVP | v2 | Описание |
|--------|:---:|:---:|----------|
| Email + пароль | ✅ | | Основной |
| OAuth Google | ✅ | | Через Firebase Auth |
| OAuth Apple | ✅ | | Требуется для App Store |
| OAuth VK ID | | ✅ | Популярен в РФ |
| Telegram Login Widget | | ✅ | Привязка к боту |
| Биометрия (Face ID / Touch ID) | | ✅ | Для быстрого входа |

### Требования к паролю

| Параметр | Значение |
|----------|----------|
| Мин. длина | 8 символов |
| Обязательно | Буквы + цифры |
| Хранение | bcrypt (cost factor 12) |
| Сброс | Email-ссылка (TTL 1 час) |

### Сессии

| Параметр | Значение |
|----------|----------|
| Токен | JWT (access 15 мин + refresh 30 дней) |
| Хранение на устройстве | Secure Enclave / Android Keystore |
| Мульти-устройства | До 5 активных сессий |
| Принудительный выход | Из настроек — отозвать все сессии |

## 2. Персональные данные (ФЗ-152 / GDPR)

### Какие данные собираем

| Категория | Данные | Основание |
|-----------|--------|-----------|
| **Идентификация** | ФИО, дата рождения, город | Регистрация на мероприятие |
| **Контакты** | Email, телефон, Telegram | Связь, верификация |
| **Спортивные** | Разряды, результаты, PB | Составление протоколов |
| **Собаки** | Кличка, порода, чип, вакцинации | Ветконтроль, идентификация |
| **Финансовые** | Статус оплаты (без платёжных данных) | Учёт регистраций |
| **Технические** | Device ID, IP, логи | Безопасность, debug |

### Согласия (Consent)

| Согласие | Когда запрашиваем | Обязательно |
|----------|-------------------|:-----------:|
| Обработка ПД | Регистрация аккаунта | ✅ |
| Обработка ПД собаки | Добавление собаки | ✅ |
| Фото/видео на мероприятии | Регистрация на мероприятие | ✅ |
| Отказ от ответственности (waiver) | Регистрация на мероприятие | ✅ |
| Push-уведомления | Первый запуск | ❌ |
| Telegram-бот | Привязка аккаунта | ❌ |
| Маркетинговые рассылки | Настройки | ❌ |

### Права пользователя

| Право | Реализация |
|-------|------------|
| **Просмотр данных** | Экспорт профиля в JSON (PR6 Настройки) |
| **Исправление** | Редактирование профиля (PR1) |
| **Удаление** | «Удалить аккаунт» → мягкое удаление 30 дней → полное |
| **Отзыв согласия** | Настройки уведомлений + отвязка Telegram |
| **Переносимость** | Экспорт результатов в CSV/PDF |

### Удаление аккаунта

```
Пользователь → «Удалить аккаунт»
  → Предупреждение: «Ваши результаты станут анонимными»
  → Подтверждение email-кодом
  → Soft delete (30 дней — возможность восстановить)
  → Hard delete:
     - ПД удаляются
     - Результаты обезличиваются (имя → «Участник #42»)
     - Собаки удаляются
     - Членство в клубах — снимается
     - Audit Log — подпись остаётся (обезличенная)
```

## 3. Безопасность клубов

### Роли в клубе

| Роль | Права |
|------|-------|
| **Владелец** | Всё + передача владения + удаление клуба |
| **Администратор** | Управление участниками, взносы, настройки |
| **Тренер** | Просмотр участников, создание мероприятий |
| **Участник** | Просмотр, регистрация на клубные мероприятия |

### Видимость данных клуба

| Данные | Публично | Участники | Админ |
|--------|:--------:|:---------:|:-----:|
| Название, город, описание | ✅ | ✅ | ✅ |
| Кол-во участников | ✅ | ✅ | ✅ |
| Список участников (имена) | ❌ | ✅ | ✅ |
| Контакты участников | ❌ | ❌ | ✅ |
| Статус оплаты взносов | ❌ | ❌ | ✅ |
| Финансовая сводка | ❌ | ❌ | ✅ |

### Приглашения и заявки

| Действие | Кто может |
|----------|-----------|
| Подать заявку | Любой |
| Принять/отклонить | Владелец, Админ |
| Пригласить по email | Владелец, Админ |
| Исключить участника | Владелец, Админ |
| Изменить роль | Владелец |
| Передать владение | Владелец (с подтверждением email) |

## 4. Безопасность мероприятий (дополнение к 07)

### Создание мероприятий

| Кто | От чьего имени | Модерация |
|-----|----------------|-----------|
| Любой пользователь | Лично | Не требуется |
| Владелец/Админ клуба | От клуба | Не требуется |
| Федерация (v2) | Официальное | Верификация статуса |

### Финансовая изоляция

- Каждое мероприятие — отдельный «кошелёк» (учёт)
- Клубные мероприятия — видимы владельцу клуба + организатору
- Нет доступа к оплатам других мероприятий

## 5. Rate Limiting и защита API

| Эндпоинт | Лимит | Действие при превышении |
|----------|-------|------------------------|
| Авторизация | 5 попыток / мин | Блокировка 15 мин |
| Верификация (email/TG) | 3 попытки / мин | Cooldown 5 мин |
| Регистрация аккаунта | 3 / час с одного IP | CAPTCHA |
| API запросы | 100 / мин | HTTP 429 |
| Загрузка файлов | 10 МБ / файл, 50 МБ / час | Reject |
