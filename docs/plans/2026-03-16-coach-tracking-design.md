# Тренерский Режим — Дизайн-документ

## Обзор

Тренерский режим — это рабочий инструмент тренера для отслеживания спортсменов на трассе во время соревнований. Тренер может встать в любой точке трассы, фиксировать прохождение интересующих его спортсменов и видеть разрывы между ними в реальном времени. После гонки — анализировать данные с графиками и историей прогресса.

### Ключевые принципы
- **Персональный хронометр** — тренер ставит отсечки только для тех спортсменов, которые ему интересны
- **Скорость фиксации** — ни одна отсечка не должна быть потеряна, даже при плотном потоке спортсменов
- **Мгновенная аналитика** — разрывы между спортсменами видны здесь и сейчас
- **Поддержка кругов** — автоматическое определение номера круга при повторной отсечке

---

## Роль тренера в системе

Тренер — это **двойная роль**:

1. **В клубе** — тренер привязан к клубу, видит список своих спортсменов, их историю результатов и прогресс от соревнования к соревнованию.
2. **На мероприятии** — тренер подключается к конкретному соревнованию как наблюдатель. Получает доступ к live-данным гонки и может открыть "Тренерский пост" — персональный хронометр.

### Точки входа
- **Из клуба** → Профиль клуба → Тренерская панель → История спортсмена
- **На мероприятии** → Детали мероприятия → "Тренерский пост" (новая кнопка)

---

## Архитектура экранов

Тренерский пост реализован как экран с **тремя полноэкранными табами**, между которыми тренер переключается свайпом или тапом по заголовку:

### Таб 1: 📊 «Гонка» (Live-таблица соревнования)

**Режим:** Read-only, реальное время.

Полноэкранная таблица с данными, которые поступают от официальных постов хронометража (Финиш, Маршалы):
- Текущее положение каждого спортсмена (кто финишировал, кто на какой отсечке)
- Официальные сплиты с маршальских чекпоинтов
- Отставание от лидера
- Статусы: DNS / DNF / DSQ

**Источники данных:** Supabase realtime или P2P mesh-сеть (если нет интернета).

Тренер **не вводит ничего** на этом табе — просто наблюдает общую картину гонки, чтобы быстро оценить обстановку.

---

### Таб 2: ⏱ «Мои отсечки» (Рабочий экран тренера)

**Режим:** Ввод данных + live-аналитика.

Это основной экран тренера. Здесь он ставит персональные отсечки и видит разрывы.

#### Два режима ввода (переключаются кнопкой вверху экрана):

##### Режим «Сетка BIB»
- Плитки с BIB-номерами всех участников соревнования
- Фильтр: «Только мой клуб» / «Все»
- Тап по плитке = мгновенная фиксация времени прохождения
- На плитке отображается:
  - BIB-номер и фамилия
  - Бейдж с номером текущего круга (①②③④⑤)
  - Время последнего круга
  - Стрелка ▲/▼ — ускорился или замедлился относительно предыдущего круга
- Цветовая индикация:
  - **Белый/нейтральный** — не прошёл через точку тренера
  - **Зелёный** — отмечен (прошёл)
  - **Жёлтый** — подозрительно большой разрыв от предыдущего круга

##### Режим «Секундомер»
- Экран превращается в одну большую кнопку **«⏱ ОТСЕЧКА»** (аналог кнопки на Финише)
- Каждый тап фиксирует время с точностью до миллисекунд
- Вибрация (haptic feedback) при каждой фиксации
- Формируется очередь неопознанных отсечек:
  ```
  ① 00:12:34.2 — [?] ← назначить BIB
  ② 00:12:35.8 — [?]
  ③ 00:12:36.1 — [?]
  ④ 00:12:38.9 — [?]
  ```
- Тренер тапает на `[?]` → открывается BIB-пикер → выбирает нужного спортсмена
- Система автоматически определяет номер круга (если BIB уже был отмечен ранее)
- Отсечки без BIB **сохраняются** — можно привязать или удалить позже
- Свайп влево по ошибочной отсечке → удалить

#### Таблица разрывов (нижняя часть экрана в обоих режимах)
Мини-таблица уже зафиксированных спортсменов с нарастающим разрывом:

```
Круг 3 из 5                    ← переключатель кругов
──────────────────────────────
#07  Петров    12:45  (▼ +9с)
#24  Иванов    12:52  (+0:07 от Петрова)  
#42  Морозов   13:18  (+0:33 от Петрова)
```

- Переключатель позволяет посмотреть разрывы на любом завершённом круге
- Цветовая индикация: зелёный = ускорился, красный = замедлился

---

### Таб 3: 📈 «Аналитика» (Post-race)

**Режим:** Полный аналитический отчёт, доступен после гонки.

#### Таблица сплитов
Полная таблица всех отсечек тренера с разрывами по каждому кругу:

| BIB | Имя | Круг 1 | Круг 2 | Круг 3 | Итого | Δ от лидера |
|-----|-----|--------|--------|--------|-------|-------------|
| 07 | Петров | 12:34 | 12:40 | 12:45 | 37:59 | — |
| 24 | Иванов | 12:37 | 12:47 | 12:52 | 38:16 | +0:17 |
| 42 | Морозов | 12:51 | 13:05 | 13:18 | 39:14 | +1:15 |

#### Графики динамики
- **График покруговых сплитов** — линия для каждого спортсмена, видно кто ускоряется, кто сдаёт
- **График нарастающего разрыва** — наглядно видно, где именно спортсмен начал терять/набирать

#### История прогресса (от соревнования к соревнованию)
- Привязка данных к профилю спортсмена
- Сравнение: «На этой дистанции Петров стабильно теряет на 2-м круге»
- Тренды: средний сплит, лучший/худший круг, динамика за сезон

#### Экспорт
- PDF-отчёт для печати
- Excel/CSV для дальнейшего анализа

## Поддержка кругов

### Автоматическое определение круга
При повторной фиксации того же BIB система автоматически увеличивает счётчик кругов:
- 1-й тап по BIB #07 → Круг 1, время 00:12:34
- 2-й тап по BIB #07 → Круг 2, время 00:25:10, сплит круга = 12:36
- 3-й тап по BIB #07 → Круг 3, время 00:37:55, сплит круга = 12:45, замедлился на +9 сек

### Отображение на плитке BIB (режим «Сетка»)
```
┌──────────────┐
│    #07       │  ← BIB-номер
│   Петров     │  ← Фамилия
│   ③ / 5      │  ← Текущий круг / Всего кругов
│  12:45 ▼     │  ← Сплит последнего круга + тренд
└──────────────┘
```

## Модель данных

### Отсечка тренера (CoachMark)
```
{
  id: UUID,
  eventId: String,         // ID мероприятия
  coachId: String,         // ID тренера (userId)
  bib: String?,            // BIB-номер (может быть null до назначения)
  athleteId: String?,      // ID спортсмена (для истории)
  timestamp: DateTime,     // Точное время отсечки
  lap: int,                // Номер круга (авто-определение)
  lapSplit: Duration?,     // Время круга (разница с предыдущей отсечкой этого BIB)
  pointName: String?,      // Название точки тренера (опционально)
  gpsLat: double?,         // GPS-координата (опционально)
  gpsLon: double?,         // GPS-координата (опционально)
  createdAt: DateTime,
}
```

### Хранение
- **Локально:** Drift (SQLite) — работает без интернета
- **Облако:** Supabase — синхронизация для истории и аналитики
- **Привязка к спортсмену:** через `athleteId` — позволяет строить тренды от соревнования к соревнованию

## Переиспользуемые компоненты

Из `FinishScreen`:
- Логика отсечек (`_marks`, `_addMark`)
- BIB-пикер (`_showBibPicker`) — с добавлением фильтра «Мой клуб»
- UI большой кнопки «ОТСЕЧКА»
- `Dismissible` свайп для удаления ошибочных отсечек
- Стиль `_glassInputDecoration`

Из `MarshalScreen`:
- Сетка BIB-номеров (`AppBibTile`)
- Логика синхронизации с Master Node

Новые компоненты:
- `CoachTimingScreen` — главный экран с 3 табами
- `CoachGapTable` — виджет таблицы разрывов
- `CoachLapBibTile` — расширенная BIB-плитка с кругами
- `CoachAnalyticsScreen` — графики и история
